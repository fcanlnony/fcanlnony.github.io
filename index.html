为什么要选择GNU/Linux

2021-08-15

关于这个问题很多朋友会问，为什么要选择GNU/Linux。可能很多人会问，不是linux吗？
在这里针对这个问题订正一下，linux是一个内核，我们日常用的是linux发行版，全名GNU/Linux。
关于我为什么会选择GNU/Linux首先就是因为兴趣使用它，在使用了一段时间后，我越发
发现我离不开它，它没有注册表之类的东西，可以很容易的安装软件，启动软件也不用找，可以
直接终端启动。可以很直观的用命令安装，卸载软件。也是从兴趣使用到离不开手。
如果是选GNU/Linux为主力系统究竟有什么好处，这个嘛，我个人认为吧，编程环境更好，
写程序也舒服，GNU/Linux的一些桌面环境拥有挺高的可玩性，比如Gnome,kde,cinnamon,xfce4等，
以及窗口管理器i3wm,dwm,bspwm,awesome等，可以把大量时间花在美化上。如果觉得现有的桌面
环境不适合您，您可以选择自己找组件定制。
GNU/Linux可以给老同志复活，如果您用lxqt，lxde这种轻量级桌面又或者窗口管理器，内存占用
可以低很多，再用上一些高性能内核linux-ck,linux-pf,linux-xanmod加上uksm，可以减少内存占用，
提高CPU性能。举个例子：我的电脑是金士顿1333频率的4G内存+奔腾G2020，放在如今，电脑是卡顿慢。
但我的系统是Gentoo，得益于Gentoo的安装软件机制，开机tty内存占用只有38MB，甚至进入dwm（美化过的）
只有134MB的内存占用。
曾经在Arch上使用deepin-wine的qq，桌面环境是Gnome，内存占用很少，甚至再开一个火狐都没有
超过2000MB的内存占用（个人电脑情况，不代表所有人）。
给新入坑的人一点建议：
选Debian系的人：可以选Ubuntu（用户量庞大，问题容易查找到解决方案），mint（基于Ubuntu，cinnamon美观）
deepin（dde挺美观，对新用户友好）
选RHEL系的人：可以选Fedora（有红帽的集成开发环境）
选Arch系的人：manjaro（对新用户友好）
追求图形界面美观的：Zorin（除了用户友好，还美观），Pop!_OS（玩游戏挺好的），Solus（Budgie挺漂亮的）
注：Linux发行版是可以随意的安装其它桌面并切换的，所以不单单局限于默认的桌面环境
如果想做点挑战：NixOS,Slackware,Arch,Linux from scratch(LFS),Gentoo都具有挑战性。NixOS,Slackware，Arch
如果想挑战可以尝试一些，这些发行版安装软件还是算容易的。LFS的话，它是一本书，可以让您深入了解GNU/Linux的原理，
您可以直接按照书上照搬建立一个发行版，也可以自己按照书上修改定制。Gentoo可以看作是用有包管理器的LFS,
它和LFS一样要编译源码，但由于有Portage可以很轻松的完成安装系统。Gentoo是个灵活的发行版，可以一个软件多版本
共存。

[START]
#准备阶段
如果是台式您可以在终端直接输入dhcpcd了！
分区的话建议提前分好区或者使用Cfdisk //有个类图形化方便分区
如果是UEFI+GPT的用户需要格式化EFI分区：mkfs.fat -F 32 /dev/sdx //sdx为您的分区，举个例子：sda1
格式化并使用swap：mkswap /dev/sdx //这步是用来初始化swap分区
swapon /dev/sdx //这步是用来激活swap分区
接下来挂载root分区：mount /dev/sdx /mnt/gentoo
如果/tmp/需要挂载独立分区：chmod 1777 /mnt/gentoo/tmp // /var/tmp同样适用
#安装stage3
验证当前时间：date
自动设置时间：ntpd -q -g
手动设置时间：date 100313162016 //100313162016只是一个例子，MMDDhhmmYYYY 语法 (月, 日, 小时, 分钟 和 年)
选择一个stage包：
multilib（32和64位）：该压缩包 multilib 尽可能使用64位的库，只必要时对32位版本兼容。
no-multilib（纯64位）：选择一个no-multilib压缩包:no-multilib是在系统的基础上提供了一个完整的64位操作系统环境。这有效地使得切换到multilib的profile是不可能的(不是完全不可能)。 //不建议刚刚开始使用Gentoo的新手使用，除非“绝对必要”。
init系统：Gentoo支持Systemd和openRC，本文为openRC的安装方法
现在先进入Gentoo的挂载点：cd /mnt/gentoo
接着进入镜像站下载：links mirrors.163.com/gentoo //请访问releases/amd64/autobuilds/ 找到amd64的镜像current-stage3-amd64/选择例如stage3-amd64-20210630T21.tar.gz的stage3包下载或者找到如20210808T170546Z（注意日期最好为新的），选择如stage3-amd64-openrc-20210808T170546Z.tar.xz的压缩包下载
接下来解压安装包：tar xpvf stage3-.tar.bz2 –xattrs-include=’.‘ –numeric-owner //tar.bz2格式解压
tar xpvf stage3-.tar.xz –xattrs-include=’.‘ –numeric-owner //tar.xz格式解压
接着启动编辑器（文中编辑器默认使用nano）：nano -w /mnt/gentoo/etc/portage/make.conf
COMMON_FLAGS=”-march=native -O2 -pipe” //可以修改-O2为-O3，使编译速度更快
第一个设置是标志 -march= 和 -mtune= ，指定目标体系结构的名称。 可能用到的选项在make.conf.example文件中有描述（作为注释）。
一个常用的值是“native”，它告诉编译器选择当前系统体系结构（用户正在安装Gentoo时的系统）。 

   第二个是标志 -O（即大写的字母O，而不是数字零），它指定了gcc优化级别标志。 可能用到级别的是s（对于大小最优化），0（零 - 无优化），1,2或甚至3等更多的优化选项（每个级别具有与前面相同的标志，加上一些额外选项）。
   -O2是建议的默认值。 -O3在整个系统范围内使用时会导致问题，因此我们建议您坚持使用-O2。

     另一个普遍使用的优化标记是-pipe（不同编译阶段通信使用管道而不是临时文件）。
     它对产生的代码没有任何影响，但是会使用更多的内存。在内存不多的系统里，gcc可能会被杀掉。如果是那样的话，就不要用这个标记。 

        使用 -fomit-frame-pointer（它将不在寄存器里为不需要帧指针的函数保存帧指针）可能会在调试程序的时候造成严重后果！

然后在make.conf最后那里补上：MAKEOPTS=”-j2” //-jx设置的数字为您的CPU数目（CPU核心数）+1 是个不错的选择，但是这个准则并不总都是最佳的。

添加源：
使用工具：mirrorselect -i -o >> /mnt/gentoo/etc/portage/make.conf
手动：GENTOO_MIRRORS=”https://mirrors.ustc.edu.cn/gentoo/" //这里我用的是中科大的源，https://mirrors.ustc.edu.cn/gentoo/ 可以替换成您喜欢的源
接下来：mkdir –parents /mnt/gentoo/etc/portage/repos.conf
cp /mnt/gentoo/usr/share/portage/config/repos.conf /mnt/gentoo/etc/portage/repos.conf/gentoo.conf

然后将sync-uri = rsync://rsync.gentoo.org/gentoo-portage 修改为 sync-uri = rsync://rsync.mirrors.ustc.edu.cn/gentoo-portage/

//警告
//当使用非Gentoo安装媒介时，这时可能还不算完。一些发行版将/dev/shm符号链接到/run/shm/，在chroot后将变得不可用。为了让/dev/shm/是一个正常挂载的tmpfs，可以这样修复：

//同时确保设置了权限为1777：
//chmod 1777 /dev/shm

#配置系统引导程序Bootloader
默认：GRUB2
emerge –ask –verbose sys-boot/grub
安装GRUB到硬盘上：
BIOS/MBR:grub-install /dev/sda //这里的a替换成您的那块硬盘
UEFI/GPT:grub-install –target=x86_64-efi –efi-directory=/boot
//注意：当/boot分区没有格式化成vfat时，必须修改 –efi-directory选项到EFI系统分区。
//UEFI用户注意：
如果 grub_install 返回了一个错误，类似 Could not prepare Boot variable: Read-only file system，那么为了成功安装，可能必须需要将 efivars 重新挂载为读写：
mount -o remount,rw /sys/firmware/efi/efivars
[W]一些主板制造商似乎只支持EFI系统分区（ESP）中.EFI文件的 /efi/boot/目录。 GRUB安装程序可以使用 –removable选项自动执行此操作。 在运行以下命令之前验证是否已安装ESP。 假设ESP安装在/boot（如前所述），执行：
[W]grub-install –target=x86_64-efi –efi-directory=/boot –removable
[W]这将创建UEFI规范定义的默认目录，然后将 grubx64.efi 文件复制到由同一规范定义的“默认”EFI文件位置。
生成GRUB配置文件：grub-mkconfig -o /boot/grub/grub.cfg
UEFI备选：rEFInd
emerge –ask –verbose app-misc/linux-logo sys-boot/refind
refind-install
//选rEFInd的原因是双系统只需要PE还原引导到EFI分区rEFInd就可以引导
更多备选方案：https://wiki.gentoo.org/wiki/Handbook:AMD64/Installation/Bootloader/zh-cn
重启系统：
[1]exit
[2]cd
[3]umount -l /mnt/gentoo/dev{/shm,/pts,}
[4]umount -R /mnt/gentoo
[5]reboot
